# AI Academy — Team Presentation

---

## OPENING

Picture this: it's Monday, 9 AM. You open Jira. There's a ticket: "Migrate the payments module to the new API v3." Before, that was 3 days of work. Today, an AI agent reads the ticket, analyzes the current code, plans the changes, implements the migration, runs the tests, and opens a PR — all before standup.

This isn't science fiction. This is what we're going to learn to build.

---

## THE PROBLEM

We've all used AI to generate code. Copy and paste from ChatGPT. Let Copilot autocomplete. But let's be honest — how many times was the result useless? How many times did we have to rewrite everything?

The problem isn't AI. The problem is we're using it like a calculator when it's an entire development team.

The difference between pasting code from a chat and orchestrating autonomous agents is the same difference between sending an email and leading a team. Both communicate — but only one scales.

---

## THE SOLUTION

AI Academy is our internal learning platform. 10 modules. 50 lessons. Fully interactive — real working code, quizzes that validate understanding, and a playground where you can watch agents thinking step by step.

The core concept you'll master is **TAOR**: Think, Act, Observe, Repeat.

This is how Claude Code solves problems:

- **Think**: What do I need to know? What could go wrong?
- **Act**: Take ONE specific action
- **Observe**: Did it work? What did I learn?
- **Repeat**: Until the problem is solved

Once you understand this cycle, you'll be able to design agents that don't guess — they *reason*.

## THE PATH

The platform has a clear learning path. Nobody gets lost.

### Phase 1 — Foundations (Modules 1-2)

How LLMs think. Prompt engineering. When to use Opus vs Sonnet vs Haiku. How an agent uses tools. How it handles errors without crashing.

### Phase 2 — Orchestration (Modules 3-5)

Multiple coordinated agents. One plans, another codes, another tests. Parallel, hierarchical, and sequential patterns. Security. Testing non-deterministic systems. Cost optimization.

### Phase 3 — The Deep Arsenal (Modules 6-10)

- **TAOR** — The pattern that unifies everything. 5 dedicated lessons.
- **LangChain** — Chains, agents, persistent memory.
- **LangGraph** — Stateful graphs, conditional routing, cycles.
- **MCP Servers** — Connect Jira, Grafana, databases, Git — all accessible to your agents.
- **RAG** — Let your agents understand your entire codebase, not just the open file.

And something no other course teaches: **how to configure CLAUDE.md** so every agent inherits YOUR team's conventions from the first turn.
