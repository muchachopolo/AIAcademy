# AI Academy — Presentacion para el Equipo

---

## APERTURA

Imaginen esto: es lunes a las 9 AM. Abren Jira. Hay un ticket: "Migrar el modulo de pagos a la nueva API v3". Antes, eso eran 3 dias de trabajo. Hoy, un agente de IA lee el ticket, analiza el codigo actual, planifica los cambios, implementa la migracion, corre los tests, y abre un PR — todo antes del standup.

Esto no es ciencia ficcion. Es lo que vamos a aprender a construir.

---

## EL PROBLEMA

Todos hemos usado IA para generar codigo. Copiar y pegar de ChatGPT. Pedirle a Copilot que autocomplete. Pero seamos honestos — cuantas veces el resultado fue inutil? Cuantas veces tuvimos que reescribir todo?

El problema no es la IA. El problema es que la estamos usando como una calculadora cuando es un equipo completo de desarrollo.

La diferencia entre pegar codigo de un chat y orquestar agentes autonomos es la misma diferencia entre enviar un email y dirigir un equipo. Ambos comunican — pero uno escala.

---

## LA SOLUCION

AI Academy es nuestra plataforma interna de aprendizaje. 10 modulos. 50 lecciones. Todo interactivo — codigo real que funciona, quizzes que validan, y un laboratorio donde pueden ver agentes pensando paso a paso.

El concepto central que van a dominar es **TAOR**: Pensar, Actuar, Observar, Repetir.

Asi es como Claude Code resuelve problemas:

- **Piensa**: Que necesito saber? Que podria salir mal?
- **Actua**: Toma UNA accion especifica
- **Observa**: Funciono? Que aprendi?
- **Repite**: Hasta que el problema este resuelto

Cuando entiendan este ciclo, van a poder disenar agentes que no adivinan — que *razonan*.

---

## EL CAMINO

La plataforma tiene una ruta clara. Nadie se pierde.

### Fase 1 — Fundamentos (Modulos 1-2)

Como piensan los LLMs. Prompt engineering. Cuando usar Opus vs Sonnet vs Haiku. Como un agente usa herramientas. Como maneja errores sin colapsar.

### Fase 2 — Orquestacion (Modulos 3-5)

Multiples agentes coordinados. Uno planifica, otro codea, otro testea. Patrones paralelos, jerarquicos, secuenciales. Seguridad. Testing de sistemas no-deterministicos. Optimizacion de costos.

### Fase 3 — El arsenal profundo (Modulos 6-10)

- **TAOR** — El patron que unifica todo. 5 lecciones dedicadas.
- **LangChain** — Cadenas, agentes, memoria persistente.
- **LangGraph** — Grafos con estado, rutas condicionales, ciclos.
- **MCP Servers** — Conectar Jira, Grafana, bases de datos, Git — todo accesible para sus agentes.
- **RAG** — Que sus agentes entiendan su codebase completo, no solo el archivo abierto.

Y algo que ningun otro curso ensena: **como configurar CLAUDE.md** para que cada agente herede las convenciones de SU equipo desde el primer turno.

## REAL WORLD

Esto no es teoria. El modulo final tiene recetas que pueden usar manana:

- Un agente que **revisa PRs** automaticamente — detecta vulnerabilidades, bugs, y sugiere fixes con codigo especifico.
- Un agente que **genera documentacion** desde el codigo — y la mantiene sincronizada.
- Un agente de **respuesta a incidentes** — analiza logs, correlaciona con deploys recientes, ejecuta runbooks, y escribe el post-mortem.

Cada receta viene con codigo listo para produccion.

---

## EL IMPACTO

Despues de completar AI Academy, cada uno de ustedes va a poder:

- Convertir un ticket de Jira en un PR revisado — sin tocar el teclado
- Disenar sistemas multi-agente para cualquier flujo de trabajo
- Saber cuando usar Opus (caro pero brillante) vs Haiku (rapido y barato)
- Implementar guardrails para que los agentes nunca hagan algo peligroso
- Medir, evaluar y optimizar sus agentes como cualquier otro sistema en produccion

No van a competir con la IA. Van a **multiplicarse** con ella.

El equipo que domina la orquestacion de agentes no escribe mas codigo.

Escribe *mejor pensamiento*.
